# Тест-кейсы для эндпоинта подписки (POST /api/subscribe)

## TC-SUB-02: Повторная подписка после истечения cooldown-периода

*   **Описание:** Проверяем, что имя подписчика обновляется, если отправить запрос с тем же email-адресом, но по истечении 30 секунд.
*   **Тип теста:** Интеграционный (API)
*   **Метод:** POST
*   **Путь:** /api/subscribe

### Предусловия
1. В базе данных в таблице subscribers есть запись с email test@gmail.com и name "John".
2. Поле last_subscribed_at у этой записи должно быть создано минимум 30 секунд назад. То есть cooldown должен пройти.

### Шаги воспроизведения
1. Запустить Postman (или аналогичный клиент).
2. Создать POST запрос на http://localhost:8000/api/subscribe.
3. Добавить заголовок Content-Type: application/json.
4. В тело запроса в формате JSON вставить:
    ```json
    {
      "email": "test@gmail.com",
      "name": "Johnny",
      "hp": ""
    }
    ```
5. Отправить запрос.

### Ожидаемый результат
1. **Ответ API:** статус 200 OK. Тело ответа выглядит так:
    ```json
    {
      "status": "success",
      "message": "Подписка успешно оформлена!",
      "data": {
        "email": "test@gmail.com",
        "name": "Johnny",
        "timestamp": "YYYY-MM-DDTHH:MM:SS.mmmmmm"
      }
    }
    ```
2. **В базе данных:** у записи с email test@gmail.com имя обновилось с "John" на "Johnny". Поле last_subscribed_at обновилось на время отправки запроса. Новые строчки в БД не создавались.
3. **Почта:** на адрес test@gmail.com ушло письмо с приветствием на имя "Johnny".
