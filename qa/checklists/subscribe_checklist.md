# Чек-лист для API подписки (POST /api/subscribe)

Этот список проверок нужен для тестирования эндпоинта подписки на новости.

## 1. Позитивные сценарии (Happy Path)

| ID | Что проверяем | Входные данные (JSON) | Ожидаемый результат (API) | Что должно измениться в БД / почте |
|---|---|---|---|---|
| TC-SUB-01 | Новая подписка | email: новый адрес (например, test@gmail.com)<br>name: "John"<br>hp: "" | HTTP 200 OK<br>Сообщение "Подписка успешно оформлена!" | Новая запись добавилась в таблицу subscribers. Ушла задача на отправку письма. |
| TC-SUB-02 | Повторная подписка после cooldown (30 секунд) | email: уже есть в БД<br>name: новое имя "Johnny"<br>hp: "" | HTTP 200 OK<br>Сообщение "Подписка успешно оформлена!" | Имя в БД обновилось на "Johnny". Поле last_subscribed_at обновилось на текущее время. |

## 2. Негативные сценарии (Валидация полей)

| ID | Что проверяем | Входные данные (JSON) | Ожидаемый результат (API) | Что должно измениться в БД / почте |
|---|---|---|---|---|
| TC-SUB-03 | Невалидный формат почты | email: "invalid-email.com"<br>name: "John" | HTTP 422 Unprocessable Entity<br>Ошибка валидации Pydantic | В БД ничего не меняется. Письмо не отправляется. |
| TC-SUB-04 | Запрос без обязательного поля email | name: "John" | HTTP 422 Unprocessable Entity<br>Ошибка валидации | В БД ничего не меняется. Письмо не отправляется. |
| TC-SUB-05 | Запрос без обязательного поля name | email: "test@gmail.com" | HTTP 422 Unprocessable Entity<br>Ошибка валидации | В БД ничего не меняется. Письмо не отправляется. |
| TC-SUB-06 | Пустой JSON | {} | HTTP 422 Unprocessable Entity | В БД ничего не меняется. Письмо не отправляется. |

## 3. Ограничения и безопасность

| ID | Что проверяем | Входные данные (JSON) | Ожидаемый результат (API) | Что должно измениться в БД / почте |
|---|---|---|---|---|
| TC-SUB-07 | Повторная подписка раньше 30 секунд (cooldown) | email: уже есть в БД<br>name: "John"<br>hp: "" | HTTP 429 Too Many Requests<br>Сообщение "Пожалуйста, подождите..." | В БД данные не меняются. Повторное письмо не уходит. |
| TC-SUB-08 | Защита от спама (заполнен Honeypot) | email: "bot@gmail.com"<br>name: "Bot"<br>hp: "any_text" | HTTP 200 OK<br>{"status": "ok"} | Запись НЕ создается в БД. Письмо НЕ отправляется. |
| TC-SUB-09 | Попытка SQL-инъекции в поле имени | name: "John'; DROP TABLE subscribers; --"<br>email: "sql@gmail.com" | HTTP 200 OK<br>Успешная подписка | Запрос отработал безопасно через ORM. Имя сохранилось в БД как строка, таблица не удалена. |
